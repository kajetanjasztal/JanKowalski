How it differs from John Doe

Break the nav

Links to the sub-pages are kept adjacent to their sections, so there is less no need to jump around the file when adding sections. This breaks accessibility quite much, since they are no longer grouped under nav tag and are scattered across the whole document.

Good luck navigating here on Lynx.

Home in the end

Home page section is moved at the end (This won't help with accessibility either). This way it can be hidden with :target~[href=#]:section, eliminating the need to position sections absolutely, since it was used there to cover the home page.

What was important

  • Keep links adjacent to their sections so adding (removing) section could be as easy as: 
    <a href="#section-id">section title</a>
<section id="section-id">
...
</section>
  • Make top-level links addressing themselves so instances of the same id are kept close together and spread across as little elements as possible.
    (Ideally one instance per id, but two in single element is not too bad).
  • Positioning them at the top of the grid helps with scroll offset (exception: sections w/o top-level link).
  • Also make section title optional when ID is good enough. So it simplifies fruther to: 
    <a id="section-id" href="#section-id"></a><section>
...
</section>
  • Keep the stylesheet minimal — don't obscure how it works with other styles.

I won't:

  • fix accessibility issues (like using nav, or moving home to the top);
  • fix the problem with padding-top in hidden sections;
  • add other section types (like .lightbox in John Doe);
  • make it pretty or ready to use (I don't think its a good idea to use it).

You can scroll up, beyond the header, but I don't care

Single-page website in html+css

This project was inspired by the John Doe (by @cadars). It's a single-page website, with multiple "sub-pages" achieved using only html and css.

Design was modified in a few ways, breaking some things and simplifying others (check out ). I've done it as a fun exercise and I don't intend to polish it in any way (maybe if it simplifies it further). I also don't encourage using it or any patterns/concepts included.

If you find it interesting as I do fill free to play with it in any way you like. And will gladly see your changes if you make some.
